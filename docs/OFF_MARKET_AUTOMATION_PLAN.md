# 🛡 StockAuto 장외시간 자동화 & 수익성 판결 시스템 구축 계획서

본 문서는 사용자의 최종 승인에 따라 작성된 **장외시간(Non-Trading Hours) 자율 자동화 및 런타임 무결성 보장 백그라운드 시스템 설계서**입니다.

---

## 📌 1. 핵심 개요 및 도입 목적

1. **24시간 '0원 서버' 200% 활용**: 거래가 없는 장외 시간(한국시간 06:00 ~ 18:00) 동안 Oracle Cloud 등 무료 서버 자원을 100% 활용하여 퀀트 연산 및 데이터 정제 수행.
2. **시뮬레이션 승률 자동 추적 & 매일 밤 최적 전략 피팅**: 정규장 개장 전 최신 미국 시장 데이터를 기반으로 72개 전략 백테스트를 자동 구동하여 최고 수익성 전략 도출.
3. **대시보드 반응 속도 0.1초화**: 백테스트 결과를 장외시간에 미리 pre-calculate하여 파일/DB 캐시에 적재함으로써 사용자가 웹에서 아레나 조회 시 0.1초 만에 즉시 표시.
4. **자동 마케팅 & SaaS 기반 조성**: 일일 매매 결산 알림 및 AI 종목 리포트 자동 발행으로 외부 사용자 유입 및 광고/구독 기반 마련.

---

## 🛡 2. 런타임 5중 무결성 및 안정성 설계 (Fail-Safe Systems)

| 구분 | 위험 요인 | 무결성 해결 솔루션 |
| :--- | :--- | :--- |
| **메모리 (OOM)** | 백테스트 연산 도중 메모리가 누적되어 웹 서버 동반 다운 | **OS 레벨 서브프로세스(`subprocess`) 완전 격리**.<br>완료 후 프로세스 종료로 점유 메모리 100% OS 강제 반환.<br>**RAM 75% 센서 가드** (75% 초과 시 백테스트 보류). |
| **네트워크** | yfinance 등 주가 API 과도 요청으로 IP 차단 또는 먹통 | **지수 백오프(Exponential Backoff)** 및 랜덤 지연(Jitter) 적용.<br>모든 요청에 **최대 30초 타임아웃** 강제 지정. |
| **중복 실행** | 이전 작업 미종료 상태에서 신규 스케줄러 도래 | **PID 락 파일 (`off_market_run.lock`)** 검증.<br>락 존재 시 0.1초 만에 안전 스킵. |
| **데이터 무결성** | 연산 도중 예외 발생으로 봇 파라미터 오염 | 연산 전 **임시 스냅샷(Snapshot)** 생성.<br>예외 발생 즉시 **기존 파라미터 100% 자동 롤백**. |
| **디스크 용량** | 로그 파일 과다 적재 | 전용 로그 파일(`logs/off_market.log`) 운영 및 7일 자동 롤오버/삭제. |

## ⏰ 3. 장외 시간대별 자동 스케줄링 시나리오 (KST 기준)

```mermaid
timeline
    title 장외시간 자율 작동 타임라인 (06:00 ~ 18:00 KST)
    06:00 ~ 09:00 : 당일 시뮬레이션 매매 결산 및 유저 알림 리포트 생성
    09:00 ~ 14:00 : 백테스트 아레나 자동 구동 & 최적 전략 파라미터 피팅 (캐시 적재)
    14:00 ~ 18:00 : AI 미장 이슈 리포트 자동 생성 및 콘텐츠 발행
```

1. **06:00 ~ 09:00 (장마감 직후)**: 당일 실현손익 정리 및 회원별 **일일 거래 일지 리포트** 생성 및 알림 발송.
2. **09:00 ~ 14:00 (낮 시간대)**: **백테스트 아레나 자동 구동**. 과거 주가 데이터를 기반으로 72개 전략 중 다음 날 최적의 승률을 낼 파라미터를 자동 피팅하여 캐시 데이터 구축.
3. **14:00 ~ 18:00 (오후 시간대)**: **AI 딥러닝 종목 분석 리포트 자동 작성**. 외부 사용자가 검색을 통해 유입될 수 있도록 미장 핫이슈 리포트를 게시판에 자동 적재.
