# Phase 14: 예수금 부족 실시간 텔레그램 경고 브릿지 및 자산 성장 차트 PoC 실장 설계도

본 설계도는 자동매매 엔진의 실거래 구동 안전성을 극대화하기 위한 '실시간 예수금 부족 텔레그램 경고 브릿지'를 구축하고, 대시보드의 비주얼 WOW 포인트를 책임질 '자산 성장 흐름 다중 차트 UI'의 PoC 안착 및 실장을 다루는 기술 설계 계획입니다.

---

## 1. 아키텍처적 특징 및 공통 설계 방향 (증권사 확장성 확보)

### A. BaseBroker 기반의 공통 예수금 방어 및 확장 메커니즘
* 현재 본 시스템은 `BaseBroker` 추상 인터페이스 하에 `SimulatedBroker` 및 `KISBroker`가 조립되어 있습니다.
* 향후 **토스증권(TossBroker) API**가 추가되더라도, `BaseBroker.get_account_balance()` 추상 메서드의 표준 규격에 맞추어 아래 데이터를 딕셔너리로 반환해주기만 하면 됩니다:
  ```python
  {
      "total_asset": int,     # 총 자산 (원화 환산)
      "cash_balance": int,    # 가용 예수금 (원화 환산)
      "stock_balance": int,   # 주식 평가금 (원화 환산)
      "profit_rate": float,   # 누적 수익률
      "provider": str         # 증권사 식별자 ("KIS Mock", "Toss Live" 등)
  }
  ```
* 이 규격을 준수하면, 백엔드 스케줄러(`scheduler.py`) 레벨의 공통 비중 조절 알고리즘 및 예수금 부족 감지 로직은 **증권사 종류에 무관하게 100% 공용 적용**되어 무결하게 동작합니다.

## 2. 백엔드: 예수금 부족 실시간 텔레그램 알림 브릿지 연동

### A. 구현 위치 및 조건
* **대상 파일**: `backend/app/bot/scheduler.py` 내 `run_user_trading_flow`
* **트리거 시점**: 매수 타점이 잡혔으나(시그널 스코어 80점 이상) 최종 안전 수량 연산 결과 가용 예수금 버짓이 모자라 1주도 매수할 수 없을 때 (`final_qty < 1`)
* **동작 시나리오**:
  1. 기존 `log_action` 경고 로그 외에 즉각 텔레그램 비동기 메시지 발송 함수 `send_message_async(user_id, message)`를 호출합니다.
  2. 사용자는 매매 타이밍을 놓치지 않도록 모바일에서 예수금 보충 즉시 즉각적인 알림을 전달받습니다.

### B. 텔레그램 경고 메시지 템플릿
```markdown
⚠️ *[자동매수 실패 - 예수금 부족]*
종목: {ticker} ({name})

• 실시간 현재가: ${current_price:.2f} (약 {current_price_krw:,.0f}원)
• 권장 투자금액: ${proposed_value_usd:.2f}
• 가용 예수금: ${cash_balance_usd:.2f}
• 최대 주문한도(95%): ${max_order_budget_usd:.2f}

안전을 위해 해당 종목 매수를 보류하고, 엔진은 Simulated(우회) 안전 동작을 유지합니다. 계좌 예수금을 점검해 주세요.
```

## 3. 프론트엔드: 자산 성장 추이 차트 PoC 실장

### A. 레이아웃 안착 위치 확정
* **대상 페이지**: `frontend/app/page.tsx` -> `Dashboard.tsx`
* **안착 자리**: `AccountBalance` 컴포넌트(계좌 현황 카드) 바로 아래, `PortfolioView` 바로 위의 1열 와이드 공간.
* **시각적 일관성**: 무채색 다크 테마(Zinc-950, Slate 계열)와 어우러지는 Vercel/Linear 스타일의 극도로 고급스러운 그라데이션 Area 차트를 구현합니다.

### B. 신규 컴포넌트 설계 (`frontend/components/AssetTrendChart.tsx`)
* **PoC 모의 데이터 연동**: 사용자의 실제 누적 `TradeLog` 히스토리 및 평가 자산 총액 흐름 데이터를 기반으로 하되, 데이터가 비어있을 경우 부드러운 우상향 곡선의 모의 자산 흐름 그래프를 렌더링하여 첫인상의 WOW 요소를 극대화합니다.
* **디자인 컴포넌트**: 무거운 외부 라이브러리 설치를 최소화하고 Next.js 15 환경에서 무결하고 가볍게 렌더링되도록 **반응형 SVG 기반의 다크 그라데이션 차트** 또는 가벼운 차트 플러그인을 활용한 프리미엄 UI로 구축합니다.

## 4. 검증 계획 (Verification Plan)

### A. 백엔드 텔레그램 알림 검증
1. 가상 예수금을 의도적으로 극도로 낮춘(예: $10) 상태로 모의 구동합니다.
2. 80점 이상의 매수 타점 종목이 탐색되었을 때, 봇 엔진이 `SKIP PURCHASE`를 트리거함과 동시에 연동된 텔레그램 봇으로 **"⚠️ [자동매수 실패 - 예수금 부족]"** 경고 메시지가 정상 수신되는지 테스트합니다.

### B. 프론트엔드 대시보드 차트 PoC 시각 검증
1. 메인 대시보드 화면에 접속하여 계좌 현황판 바로 밑에 자산 성장 추이 그래프가 미려하게 렌더링되는지 확인합니다.
2. `$ USD | 원 KRW` 세그먼트 컨트롤 버튼을 조작했을 때 차트 내 지표 및 축 단위가 동적으로 포맷팅되며 반응형으로 동작하는지 시각적 디테일을 정밀 검토합니다.
