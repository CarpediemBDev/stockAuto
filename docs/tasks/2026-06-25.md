## 1. 모의투자 수익률 환율 왜곡 보정
- **상태:** `[x]`
- **작업 설명:** `LocalSimulatedBroker.get_account_balance()`에서 미투자 현금의 환율 변동분이 수익률로 반영되는 문제를 제거하고, 수익률은 외화 기준으로 환율 효과를 제외해 관리자 수익률 스냅샷이 계산되도록 회귀 테스트를 보강한다.

### 변경 영향 기록
- **변경 분류:** 데이터 흐름, 백그라운드 스냅샷 계산
- **생산자:** `backend/app/bot/simulated_broker.py`의 `LocalSimulatedBroker.get_account_balance`
- **소비자:** `backend/app/bot/scheduler.py`의 `admin_balance_cache_sync`, `backend/app/admin/router.py`의 사용자 목록 수익률 표시
- **데이터 경계:** 사용자별 `user_id` 기준 `Holding`, `TradeLog`, `AccountEquitySnapshot` 계산 경계 유지
- **API 계약:** 응답 필드명 변경 없음. `profit_rate`, `profit_loss`, `total_asset`은 외화 기준 거래 손익률로 정상화하고, 미투자 현금 및 환율 변동 효과를 수익률에서 제외
- **문서 반영:** 장기 문서 변경 없음 - API/DB 스키마 변경 없이 계산식 보정과 회귀 테스트 추가
- **회귀 테스트:** 미거래 사용자 환율 변동 수익률 0%, 동일 USD 손익에서 환율만 바뀌어도 수익률이 고정되는 테스트 추가
- **미해결 위험:** 기존 DB에 이미 저장된 과거 왜곡 스냅샷은 별도 데이터 보정 작업 없이는 자동 수정되지 않음
- **인수인계:** `backend/app/bot/simulated_broker.py`에서 모의투자 수익률을 외화 기준으로 계산하도록 수정 완료. 다음 작업자는 과거 `AccountEquitySnapshot` 데이터 보정이 필요하면 별도 마이그레이션/정리 스크립트 범위로 시작할 것.
