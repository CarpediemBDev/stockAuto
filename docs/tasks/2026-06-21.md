### 감사 결과
- `/scanner/swing-predict` 및 갱신 API는 인증 사용자와 DB 세션을 받지만 실제 캐시는 `GLOBAL_SWING_POOL` 하나만 사용한다. 프론트엔드 툴팁의 "관심종목과 기본 주도주 풀" 계약과 불일치한다.
- `/account/balance`의 `focused_radar_tickers`는 관심종목을 강제 포함하도록 설계된 `get_focused_tickers()`에 공용 시장 신호만 전달하여 사용자 관심종목 결합이 무력화된다.
- 관심종목 CRUD, 거래·활동 로그, 개인 리포트, 보유종목 및 일반 사용자 설정 경로에서는 현재 사용자 `user_id` 필터 적용을 확인했으며 동일 유형의 직접적인 사용자 간 데이터 노출은 발견하지 못했다.
- `AGENTS.md`에는 일자별 현황판 등록과 React 전용 가이드 현행화 규칙만 있고, 아키텍처·데이터 흐름·API 계약 변경 시 관련 기술 문서를 갱신하는 일반 규칙은 없다.
- `docs/FILTER.md`와 `docs/SYSTEM_MANUAL.md`는 관심종목이 공용 스캔 시드에 포함된다고 설명하여 현재 멀티테넌시 구조와 불일치한다.
- 이번 감사에서는 코드 및 일반 지침 문서를 수정하지 않고 발견 사항을 GitHub #8에 기록했다.
