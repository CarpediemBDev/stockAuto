## 3. 토스증권 거래 모드 지원 범위 차단
- **상태:** `[R]`
- **작업 설명:** 개인 투자 설정 화면에서 토스증권 선택 시 StockAuto가 검증하지 않은 MOCK/REAL 거래 모드를 선택·저장·실행하지 못하도록 브로커별 지원 모드 계약을 정리한다.

### 변경 영향 기록
- 변경 분류: [Safety] 브로커별 거래 모드 capability 계약 및 UI/API 검증 보강
- 생산자: `backend/app/bot/broker_factory.py`의 브로커 카탈로그와 `get_broker_client`, `backend/app/admin/router.py`의 설정 저장·인증 검증 API
- 소비자: `frontend/app/admin/settings/page.tsx` 개인 투자 설정 화면, 자동매매 브로커 팩토리, 관리자 설정 API 사용자
- 데이터 경계: 사용자별 `UserSettings.trade_mode`, `UserSettings.broker_provider`, `BrokerCredential.verified_trade_mode` 조합 검증. 기존 저장값은 자동 변경하지 않고 저장·실행 진입에서 차단한다.
- API 계약: `/api/v1/admin/settings` 응답의 `available_brokers.supported_modes`가 실제 지원 가능 모드만 노출되며, 미지원 조합 저장·인증 검증은 400 오류로 차단한다.
- 문서 반영: 당일 현황판 및 장기 시스템 문서의 브로커 확장 설명을 실제 지원 범위와 맞춘다.
- 회귀 테스트: 브로커 카탈로그 단위 테스트, admin 라우터 py_compile, 프론트 lint/typecheck, 전체 verify_harness 통과
- 미해결 위험: 토스증권의 공식 실전/모의 지원 정책이 바뀌면 capability 값을 재검토해야 한다.
- 인수인계: 토스증권은 현재 StockAuto 내에서 `SIMULATED`만 허용하고, KIS는 기존 3모드 지원을 유지한다. 사용자 최종 승인 전까지 `[R]` 상태를 유지한다.
